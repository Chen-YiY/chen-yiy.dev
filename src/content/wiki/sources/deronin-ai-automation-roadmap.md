# 如何在 6 个月内成为 AI 自动化工程师

## 摘要

10000+ 字的 AI 自动化工程师 6 个月职业路线图。核心主张：**你不需要成为开发者、不需要学机器学习、不需要微调模型**——你需要的是把 AI 接到公司已在用的工具上，把重复性人工任务自动化。推荐工具：n8n（开源、免费额度大、AI 节点强、可自托管）。提供从基础工具到 Agent 使用判断、从部署监控到定价策略的完整路径。

## 关键要点

### 角色定义

AI Automation Builder 介于四种角色之间：
- **工作流自动化**：n8n、Make、Zapier
- **应用型 AI**：LLM、提示词、简单 agents
- **业务流程咨询**：知道哪些值得自动化
- **轻量技术胶水工作**：读文档、修小问题、排查 bug

**核心洞察**："他们不需要研究员，不需要微调工程师。他们需要的是：有人能走进去，找到流血点，然后用一个能在第一个月就回本的工作流把它止住。"

### 两条路径

- **非技术路径（默认）**：无代码工具（n8n）+ AI 节点，一行 Python 都不写也能构建并卖出真实自动化
- **开发者路径（可选）**：补上 Python、LangGraph 和自定义后端能力，接更大合同

**唯一规则**："选一条路，然后别再来回切换。"

### 6 个月路线图

| 月份 | 主题 | 核心技能 |
|------|------|----------|
| 第 1 个月 | 基础工具、API、提示词 | n8n 深入 + API/Webhook 概念 + Prompting |
| 第 2 个月 | AI 放进真实工作流 | AI 节点 + RAG 基础 + 结构化输出 + 错误处理 |
| 第 3 个月 | 做能反复卖的业务工作流 | Lead qualification bot + 内容自动化 + 定价与销售 |
| 第 4 个月 | 什么时候该用 Agent | Agent vs 固定工作流判断 + Human-in-the-loop + 工具描述设计 |
| 第 5 个月 | 部署、监控、文档与交付 | Railway 部署 + 执行日志 + Prompt 版本管理 + 安全基础 |

### Month 1 重点

- **工具选择**：n8n（推荐）> Make（视觉精美）> Zapier（仅 MVP）
- **学习资源**：n8n Official Docs、n8n Academy、Productive Dude (YouTube)、Nick Saraev (YouTube)
- **核心概念**：触发器 vs 动作、数据流、错误处理、HTTP Request 节点

### Month 2 重点

- AI 不再手动用 ChatGPT，而是在工作流中自动运行
- 结构化输出（JSON mode）
- 错误处理和重试逻辑
- 开发者路径：Python 基础（变量、循环、requests、try/except）

### Month 3 重点

- 做"可重复销售"的标准化自动化
- 首单建议：Lead qualification bot ($300-$500)、内容自动化 ($500-$1,500)
- 不是卖"咨询"或"战略"，而是卖具体结果

### Month 4 重点

**"网上 70% 的 AI agent 内容都是炒作。绝大多数时候，agent 都不是最优解。"**

判断框架：
- **单次 LLM 调用**：输入明确、输出可预测 → 直接调用
- **固定链**：步骤确定、不需要自主决策 → Chain/Pipeline
- **Agent**：步骤不确定、需要根据中间结果调整 → 才用 Agent

关键能力：
- 写出在不同输入下都能被正确选择的工具描述
- 给不可逆工作流加 human-in-the-loop 审批
- "面对客户时，敢于说'这个地方上 agent 反而是过度设计'"

### Month 5 重点

- **部署**：Railway 一键部署 n8n（不要过早学 Docker/K8s）
- **监控**：Better Stack（正常运行时间）+ n8n 内置执行日志 + Langfuse（LLM 可观测性）
- **Prompt 版本管理**："你的 prompts 就是代码。直接在生产工作流里随手改 prompt，是导致系统'悄悄坏掉'的最快方式。"
- **安全**：OWASP LLM Top 10、环境变量存 secrets、凭据系统

### 市场数据

- 全球 3.6 亿家公司，只有 5000 万做了部分自动化
- 还有 3.1 亿家公司未应用任何自动化
- 真正能提供这种服务的人，全世界可能只有 100 万

## 前提与边界

- **前提假设**："全球 3.1 亿家公司未应用任何自动化"的市场数据来源未明确标注出处，可能为估算值；"真正能提供这种服务的人全世界可能只有 100 万"缺乏可验证的数据基础
- **数据可靠性**：本文为社区教程整理，非学术研究或行业报告。定价参考（$500-$5,000/月）来自特定市场（推测为北美），在中国、东南亚等市场的定价可能需要大幅调整
- **不适用场景**：6 个月路线图假设学习者有稳定的网络环境和持续的实践时间，对网络受限地区（需频繁翻墙访问海外服务）存在额外门槛；n8n 作为推荐工具的优势建立在"开源+免费额度大"的前提上，企业级合规需求（数据驻留、审计）可能需要其他方案
