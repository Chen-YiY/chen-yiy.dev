# PRD: 个人技术博客 + 作品展示网站

## Context

参考 [lovstudio.ai](https://lovstudio.ai) 的卡片式作品展示风格，构建一个个人技术博客 + 开源项目展示网站。目标是将个人品牌、技术文章、开源作品整合到一个统一的暖白色调站点中。

**参考站核心特征**：
- 暖白色调背景（#f9f8f6），简洁现代，温润质感
- 卡片式项目展示墙
- 多维度分类筛选（平台/类别）
- 每个 app 卡片含：名称、描述、Star/Fork/下载量、更新时间、标签

## 3. 核心功能模块

### 3.1 项目展示模块 (`/projects`)

**参考 lovstudio.ai 的核心交互**：

```
┌─────────────────────────────────────────────┐
│  Hero: "[Username]'s Projects"              │
│  "Every project tells a story"              │
├─────────────────────────────────────────────┤
│  筛选栏:                                    │
│  [全部] [Web端] [桌面端] [CLI] [插件] ...   │
│  排序: [Star↓] [更新时间↓] [名称A-Z]       │
├─────────────────────────────────────────────┤
│  ┌──────────┐ ┌──────────┐ ┌──────────┐    │
│  │ 项目名    │ │ 项目名    │ │ 项目名    │    │
│  │ 描述...   │ │ 描述...   │ │ 描述...   │    │
│  │ ⭐ 215   │ │ ⭐ 2117  │ │ ⭐ 91    │    │
│  │ 🔄 0     │ │ 🔄 284   │ │ 🔄 9     │    │
│  │ 📥 762   │ │ 📥 392   │ │ 📥 93    │    │
│  │ 1w ago   │ │ 3w ago   │ │ 2mo ago  │    │
│  │ [标签]   │ │ [标签]   │ │ [标签]   │    │
│  └──────────┘ └──────────┘ └──────────┘    │
│         ...更多项目卡片...                   │
└─────────────────────────────────────────────┘
```

**数据字段**：
- 项目名称、描述
- Star 数、Fork 数、下载/克隆量
- 最后更新时间（相对时间）
- 标签：平台类型（Web/桌面/移动/插件）+ 功能类别
- Featured 标记（置顶推荐）
- GitHub 仓库链接

**数据来源**：GitHub REST/GraphQL API
- `GET /users/{username}/repos` 获取仓库列表
- `GET /repos/{owner}/{repo}` 获取详情
- 通过 GitHub API 的 `topics` 字段实现标签筛选
- 本地 JSON 文件补充自定义字段（Featured、中文描述等）

**交互功能**：
- 按平台/类别 Tab 筛选
- 按 Star / 更新时间 / 名称排序
- 点击卡片跳转 GitHub 仓库或项目详情页
- 响应式布局：桌面 3 列 → 平板 2 列 → 手机 1 列

### 3.2 博客模块 (`/blog`)

**文章列表页**：
```
┌─────────────────────────────────────────────┐
│  博客                                        │
│  [搜索框...]                                 │
│  标签: [全部] [安全] [前端] [Python] [DevOps] │
├─────────────────────────────────────────────┤
│  ┌──────────────────────────────────────┐    │
│  │  文章标题                              │    │
│  │  摘要文字...                           │    │
│  │  2026-04-08 · 安全 · 5 min read      │    │
│  └──────────────────────────────────────┘    │
│  ┌──────────────────────────────────────┐    │
│  │  文章标题                              │    │
│  │  ...                                   │    │
│  └──────────────────────────────────────┘    │
└─────────────────────────────────────────────┘
```

**文章详情页 (`/blog/[slug]`)**：
- MDX 渲染，支持 React 组件嵌入
- 代码块增强：语法高亮（Shiki）+ 复制按钮 + 行号 + 文件名标注
- 目录导航（Table of Contents）侧边栏
- 文章元信息：发布日期、标签、预计阅读时间
- 上一篇/下一篇导航
- Giscus/Disqus 评论（可选，Phase 2）

**博客数据**：
- 文件位置：`src/content/blog/` 目录下 `.mdx` 文件
- Frontmatter schema：
  ```yaml
  ---
  title: "文章标题"
  date: 2026-04-08
  tags: ["安全", "前端"]
  summary: "文章摘要"
  draft: false
  ---
  ```

## 4. 技术架构

### 4.1 项目结构

```
src/
├── components/          # React 组件
│   ├── layout/          # Header, Footer, Navigation
│   ├── projects/        # ProjectCard, ProjectFilter, ProjectGrid
│   ├── blog/            # BlogCard, BlogSearch, TagFilter, TableOfContents
│   ├── about/           # SkillCard, SocialLinks
│   └── ui/              # 通用 UI 组件 (Button, Badge, SearchInput)
├── content/
│   ├── blog/            # MDX 文章文件
│   │   ├── post-1.mdx
│   │   └── post-2.mdx
│   └── config.ts        # Content collections schema
├── data/
│   └── projects.json    # GitHub 数据补充（Featured 标记等）
├── layouts/
│   └── BaseLayout.astro # 基础布局
├── pages/
│   ├── index.astro      # 首页
│   ├── projects.astro   # 项目展示
│   ├── blog/
│   │   ├── index.astro  # 博客列表
│   │   └── [...slug].astro # 博客详情
│   └── about.astro      # 关于页面
├── styles/
│   └── global.css       # 全局样式 / Tailwind 配置
├── lib/
│   ├── github.ts        # GitHub API 封装
│   └── utils.ts         # 工具函数
└── config.ts            # 站点配置（个人信息、导航等）
```

### 4.2 技术选型

| 层面 | 技术 | 用途 |
|------|------|------|
| 框架 | Astro 5.x | 静态站点生成，岛屿架构 |
| UI | React 19 | 交互组件（筛选、搜索） |
| 样式 | Tailwind CSS 4.x | 暖白色调，响应式设计 |
| 代码高亮 | Shiki | 博客代码块语法着色 |
| 内容 | Astro Content Collections | MDX 文章管理 |
| 数据 | GitHub REST API | 项目数据自动同步 |
| 图标 | Lucide React / Astro Icon | 统一图标库 |
| 字体 | Inter / JetBrains Mono | UI 字体 + 代码字体 |

### 4.3 数据流

```
构建时:
  GitHub API ──→ lib/github.ts ──→ 静态 JSON ──→ 项目卡片渲染
  MDX 文件 ──→ Content Collections ──→ 博客页面生成

运行时 (Hydrated Islands):
  筛选/排序 ──→ 客户端 React 组件 (filter/sort state)
  搜索 ──→ 客户端模糊搜索 (Fuse.js 或简单 filter)
```

## 5. 视觉设计规范

### 5.1 色彩系统（暖白色调，参考 lovstudio.ai）

```css
/* 暖白背景 + 柔和灰层次 */
--bg-primary:     #f9f8f6    /* 页面背景（暖白） */
--bg-secondary:   #ffffff    /* 卡片背景（纯白） */
--bg-hover:       #f0efed    /* 悬浮态 */
--border:         #e8e6e3    /* 边框（柔和灰） */
--text-primary:   #1a1a1a    /* 主文字（深色） */
--text-secondary: #525252    /* 次要文字 */
--text-muted:     #8c8c8c    /* 辅助文字 */
--accent:         #3b82f6    /* 强调色 (蓝) */
--accent-hover:   #2563eb    /* 强调色悬浮 */
--badge-bg:       #f0efed    /* 标签背景（浅灰） */
--badge-text:     #525252    /* 标签文字 */
--shadow:         rgba(0,0,0,0.06) /* 阴影（极淡） */
```

### 5.2 响应式断点

| 断点 | 宽度 | 卡片列数 |
|------|------|----------|
| 手机 | < 640px | 1 列 |
| 平板 | 640-1024px | 2 列 |
| 桌面 | > 1024px | 3 列 |

### 5.3 设计原则

- **简洁克制**：参考 lovstudio.ai 的信息密度，不多不少
- **暖白温润**：#f9f8f6 底色 + 白色卡片 + 极淡阴影，营造温润质感
- **信息优先**：项目卡片突出 Star/Fork 等关键指标
- **微交互**：卡片悬浮微抬 + 阴影变化，筛选切换动画
- **高可读性**：深色文字在暖白底上，对比度充足

## 6. 实施计划

### Phase 1: 基础骨架（MVP）

**目标**：可访问的暖白色调站点，含基础页面

1. 初始化 Astro + React + Tailwind 项目
2. 配置暖白色调全局样式（#f9f8f6 背景）
3. 实现 BaseLayout (Header + Footer)
4. 实现首页骨架 (Hero + 占位)
5. 配置 Content Collections (博客 schema)
6. 部署到 Vercel/Cloudflare Pages

### Phase 2: 项目展示

**目标**：完整的 GitHub 项目展示 + 筛选功能

1. 封装 GitHub API (`lib/github.ts`)
2. 实现项目数据获取 + 本地 JSON 补充
3. 实现 ProjectCard 组件
4. 实现 ProjectGrid + 筛选/排序交互
5. 实现平台/类别 Tab 筛选
6. 响应式适配

### Phase 3: 博客系统

**目标**：完整的 MDX 博客阅读体验

1. 创建示例 MDX 文章
2. 实现博客列表页 + 标签筛选
3. 实现博客详情页 (MDX 渲染)
4. 配置 Shiki 代码高亮 + 复制按钮 + 行号
5. 实现 Table of Contents 侧边导航
6. 实现搜索功能

### Phase 4: 关于页面 + 打磨

**目标**：完善个人页面，优化体验

1. 实现关于页面（个人介绍 + 技能 + 社交链接）
2. 首页整合（精选项目 + 最新文章）
3. SEO 优化（meta、Open Graph、Sitemap）
4. 性能优化（图片优化、预加载）
5. RSS 订阅支持
