# i18n 中英文切换设计

## 概述

为站点添加中英文 UI 切换功能。范围：导航、按钮、标签等 UI 文本 + About 页面双版本内容。博客和 Wiki 内容保持原语言不变。

## 方案

Astro 静态翻译文件 + 构建时双 HTML 元素 + React 切换器。无 URL 路径前缀，通过 localStorage 存储语言偏好。

## 翻译文件

```
src/i18n/
  en.json    # 英文 UI 文本
  zh.json    # 中文 UI 文本
```

键名统一英文，按页面/区域分组：

```json
{
  "nav": { "projects": "Projects", "wiki": "Wiki", "blog": "Blog", "about": "About" },
  "hero": { "tagline": "Personal Tech Blog & Open Source Projects" },
  "projects": { "title": "Projects", "sort": "Sort by", ... },
  "blog": { "title": "Blog", "readMore": "Read more" },
  "wiki": { "title": "Wiki", "filter": "Filter" },
  "about": { ... },
  "footer": { ... }
}
```

`src/config.ts` 中的语言相关字段（`description`、`nav[].label`）迁移到翻译文件。Config 只保留 `author`、`github.username`、`social`。

## 语言切换机制

### 存储

- `localStorage` 存 `lang` 值：`"zh"`（默认）或 `"en"`
- 首次访问默认中文

### 切换器组件

- 新建 `src/components/LangToggle.tsx`（React）
- 导航栏右侧，紧挨 ThemeToggle
- 显示 `中 | EN`，当前语言高亮
- 点击时：更新 localStorage + 设置 `document.documentElement.lang` + 派发 `lang-change` 自定义事件

### Astro 组件响应

- 构建时，每个需要翻译的文本生成两个 `<span>`，分别带 `data-lang="zh"` / `data-lang="en"`
- BaseLayout 底部内联 `<script>`：监听 `lang-change`，按 `data-lang` 显隐元素
- 页面加载时立即从 localStorage 读取偏好，同步设置显隐（防闪烁）
- `<html lang="...">` 从 localStorage 动态设置，不硬编码

### About 页面双版本

About 页面内容较长，直接在 `about.astro` 内写两个 section：

```astro
<section data-lang="zh">
  <!-- 中文版完整内容 -->
</section>
<section data-lang="en" style="display:none">
  <!-- 英文版完整内容 -->
</section>
```

共享同一套 `data-lang` 显隐逻辑，无需额外代码。

## 不做的事

- 博客和 Wiki 内容不翻译
- 不引入 URL 路径前缀（如 `/en/about`）
- 不引入第三方 i18n 库（如 astro-i18next）
- 不为 SEO 生成 hreflang 标签（个人站点不需要）
